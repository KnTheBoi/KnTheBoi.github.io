Summary

My initial approach was a dark-theme website that has multiple bult-in functions, as I read through WebAss bits by bits, I realized there were too many prohibitions that limit lots of what I already had in mind. I understand that it is for the sake of simplicty, which is why I switched to create a portfolio-style website. There have been at least 10 prototypes, as each one pass by, I also believed my knowledge and practical-programming skill in HTML and CSS has also gotten better.

Technical Aspects

Navigation

The idea here is a sticky navigation that sticks to the top of the website. Any technical implementations here should be fairly simple. With the use of flex and justify-content, the layout of the navigation bar is neat and tidy, evenly-spaced between buttons. Although there are still some unpleasant issues with it, I personally believed that it should be suffice. If you look carefully, there should be some subtle features, current page highlighted by an underline dash, the font-size is slightly bigger, there is also the implementation of transition-duration in which the button change color on :hover

Home/Index Page

For the layout of this page, as I already mentioned, it is supposed to be a simple personal portfolio-website, so I went with flex layout, flex-direction: column as flex is great with one-dimensional axis layout, this is also applied to the other three pages, including this one. Well first of, we have multiple containers that group all child elements together, and for each child element, I can work on the y or x-axis however I want. There are pictures, scaled to be dynamic, background is an image with fixed attachment, giving it a mesmerizing feel when scrolling. And then there is a ubuntu/linux command prompt, I thought this would be a really cool idea, to capture the essence of what I do: programming, this can also be upgraded later on to be more interactive, maybe functional, maybe allowing user to input something like cat certain_file.txt and it would switch out new texts. I also implemented some GIFs, I hope it's not too much.

Past Page

As there is no specific requirement for the Future and Past Page, I used a simple layout for both. For the 'Past' page, there is a twist with the banner img/GIF, as it will change at certain resolution, this was initially a fix to the GIF resolution as it was not big enough to stretch out for the desktop resolution, and I decided to make it a feature afterwards.

Future Page

It uses the same layout as 'Past' page.

Aesthetics Aspects

Clarity, Navigability and Visual Appeal

As mentioned, I used a flex and sticky navigation bar on top for navigation. Simple and easy to use, with interactive buttons and also stationary, no need to scroll up and down.
The visual appeal ways I implemented can be found through subtle details, like a pseudo-animation for the navigation buttons, interactive images. The navbar and footer are styled with shadow-box, the websites use dark theme similar to the "MS Command Prompt". Behind the background-image I also made the color black to reduce flashing an blinking when moving to another page.
There are texts and titles for every section, so it should be easy to figure out where you are at.

Readability

For the font, I interchange between 'Comic Sans MS' and 'Consolas'. Comic Sans MS is just a personal preference, used to replace Consolas to prevent it from being over used and to enhance readability as Consolas is not too great at it. As for 'Consolas', it is a choice in order to keep the theme, idea and spirit of a programmer, of course the font-size has been set to be readable and visually appealed.
As the pages are dark theme, I went with the 'beige' color on some texts as pure white would have been too bright. There are also some orange-highlighted text to stay inline with the color scheme 'Black-White-Orange'.

Responsiveness

This was the most annoying aspect, cost me 3 prototypes, but @media queries and flex came to save the day!
I had made sure to make the websites dynamic by setting width-height with vw-vh(% to screen size) and combined it all with the flex-layout, all together with settings for 2 different breakpoint of resolution. And the websites are now responsive!

Accessibility

The webpages are accessible and cater to all types of user. Even to those who are color-blinded or vulnerable to seizures. As the two main color are white and black, both of which are mono colors, there should be no problem for anyone in reading the texts. The websites also have subtle features as mentioned to ease the any known flashing or blinking too.
